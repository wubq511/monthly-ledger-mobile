# 记账页键盘适配与概览页去说明化设计

## 背景

当前记账页为了让保存按钮始终可见，使用了底部悬浮保存区，但在安卓键盘弹起后会遮挡较大面积的输入区域。数据管理模块也直接嵌在表单下方，影响“快速记一笔”的主流程。概览页中仍有较多解释性文案，占用有限屏幕高度，导致本月账单区域没有充分利用可见空间。

## 目标

- 让记账页在键盘弹起时仍然容易保存，但不遮挡大量输入区域。
- 把数据管理改成顶部单按钮入口，并在需要时以浮窗方式展示。
- 删除概览页中多余说明性文字，让重点信息更直接。
- 让概览页账单区域在内容较少时尽量延展到导航栏上方，避免留下尴尬空白。

## 非目标

- 不修改备份/恢复的底层逻辑。
- 不调整数据库结构。
- 不改变概览页和趋势页的主导航结构。

## 交互设计

### 记账页保存按钮

记账页保存入口分为两种状态：

- 键盘收起时：保留底部主操作按钮，但去掉包裹它的外框卡片，只保留按钮本体，避免与导航栏边框相撞。
- 键盘弹起时：将保存按钮收缩成更轻量的悬浮按钮，贴近底部安全区域显示，减少对表单内容的遮挡面积。

这样既保留保存动作的可达性，也避免大面积压住金额、大类和细分区域。

### 数据管理入口

数据管理入口移动到记账页顶部 Hero 卡片下方，只保留一个按钮：

- 按钮文案：`账单备份与导入`

点击后打开一个半透明遮罩层上的居中浮窗，浮窗内部复用现有 `BackupActions` 模块内容。点击浮窗外区域关闭弹窗。

这个设计把“账本维护”从主录入路径中移开，但仍然保持发现成本低。

### 概览页去说明化

删除以下说明性文字：

- Hero 卡片中 “2026年4月 花得最多的是...”
- Hero 卡片中预算条下方的 “距离预算上限还剩...”
- “本月分类消费排名” 下方说明
- “本月账单” 下方说明

保留标题、金额、预算状态与列表本身，使页面更像工作面板，而不是带注释的报告。

### 概览页账单区域延展

将“本月账单”区域改为一个完整卡片，并让其在内容较少时具备最小高度与伸展能力：

- 当账单较少时，卡片仍向下延展，尽量贴近底部导航上方
- 当账单较多时，继续随页面滚动

这样可以减少页面下半段的空白，同时保持导航不被遮挡。

## 代码设计

### 记账页

在 `ExpenseForm.tsx` 中增加：

- 键盘显示状态监听
- 备份入口按钮状态
- `Modal` 浮窗结构

保存按钮布局根据 `keyboardVisible` 切换不同样式，而不是始终占用一个固定的大底部区域。

### 概览页

在 `App.tsx` 与展示组件中：

- 把说明性文案设为可选并在概览页关闭
- 将“本月账单”改成带边框的面板，并使用 `flexGrow`/`minHeight` 在短内容下占满更多可见区域

## 测试策略

- 保留并扩展已有图表布局测试，确保预算标签不会再次贴边溢出
- 继续通过 `npx tsc --noEmit` 与 `npm test` 验证 UI 改动没有破坏现有逻辑层
