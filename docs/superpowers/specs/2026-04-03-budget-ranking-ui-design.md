# 月度账本预算与排名增强设计

## 背景

当前应用已经支持按月份记账、总支出统计、分类汇总和月趋势展示，但还缺少明确的预算视角，也没有把“这个月钱花在了哪里”和“某个分类在哪几个月花得最多”表达得足够直观。

本轮增强聚焦两个方向：

1. 以 `2000` 元作为统一月预算，补全超支统计与预算健康表达。
2. 在不破坏现有月账本流程的前提下，增强分类排名与整体视觉层级。

## 目标

- 显示每个月是否超支，以及超支或结余金额。
- 显示所有月份累计超支总和。
- 显示平均每月超支金额，口径为 `所有超支总和 / 已记录月份数`。
- 在选中月份下展示分类消费排名。
- 在同一分类下展示不同月份的消费排名。
- 微调界面，使预算状态、金额层级和图表信息更易读、更美观。

## 非目标

- 不引入登录、云同步、预算提醒推送或后端服务。
- 不做分类预算体系，预算仍为全局统一的 `2000` 元月上限。
- 不修改“月份账本”定位，不恢复按具体日期展示。

## 推荐方案

采用轻量增强方案：继续沿用现有 React Native + Expo + SQLite 架构，把预算与排名能力并入现有汇总层，再在概览页和趋势页中以更清晰的视觉层级呈现。

这样做的原因：

- 风险低：不需要改动已有数据模型和主导航结构。
- 一致性强：所有预算、趋势、排名都从同一份月汇总数据推导，避免多个页面出现不同统计口径。
- 可扩展：后续如果要增加“饮食预算”“接近超支提醒”等能力，可以继续复用同一套汇总快照。

## 数据与统计设计

### 预算常量

- 统一月预算常量：`2000`

### 月汇总快照

在现有 `ledgerSummary` 汇总层中，为每个已记录月份生成一个快照对象，至少包含：

- `monthKey`
- `total`
- `remaining`
- `overspend`
- `isOverBudget`
- `rankedCategories`

其中：

- `remaining = max(2000 - total, 0)`
- `overspend = max(total - 2000, 0)`
- `rankedCategories` 按金额从高到低排列，金额相同时按分类名稳定排序

### 全局预算统计

基于所有月快照额外生成：

- `trackedMonthCount`：已记录月份数
- `overspendMonthCount`：发生超支的月份数
- `totalOverspend`：所有月份超支总和
- `averageMonthlyOverspend`：`totalOverspend / trackedMonthCount`
- `selectedMonthBudgetStatus`：选中月份的预算状态，包括总支出、超支、结余、预算使用率

### 分类跨月排名

增加一个按分类聚合的跨月索引：

- `categoryMonthRanking[category] = [{ monthKey, total }, ...]`

排序规则：

- 先按该分类在该月总金额从高到低排序
- 金额相同时按月份从近到远排序

默认展示策略：

- 优先展示当前选中月份“分类消费排名”中的第 1 名分类的跨月排名
- 同时提供分类切换入口，允许用户改看其他分类

这个默认值避免页面初次进入时空白，也能把“本月最花钱的分类”自然延展到跨月对比。

## 界面设计

### 概览页

概览页保留“本月总支出”为主视觉，但增加更明确的预算语义：

- 主金额区继续展示当前月份总支出
- 主金额区下方新增预算进度条，展示距离 `2000` 元上限还有多少，若超支则进入警示态
- 原有两枚摘要指标调整为更有决策价值的预算指标：
  - `本月预算状态`：显示“结余 X 元”或“超支 X 元”
  - `平均每月超支`：按所有已记录月份平均
- 新增一枚辅助指标：
  - `累计超支`

分类区做两处增强：

- `本月分类消费排名`：按金额展示当前月份各分类排行
- 保留现有分类总览，但更突出前 3 名，弱化尾部分类的视觉噪音

### 趋势页

趋势页继续承担“跨月理解”的职责，但预算信息更明确：

- 月趋势图增加 `2000` 元预算基准线
- 超支月份的点、柱或标签使用警示色
- 指标摘要区增加：
  - `超支月份数`
  - `累计超支`
  - `平均每月超支`

新增 `同分类跨月排名` 模块：

- 标题区显示当前选中的分类
- 内容按月份列出该分类在哪些月份花费最高
- 列表项展示月份、金额、排名序号

### 视觉微调方向

视觉优化遵循“更精致但不过度装饰”的原则：

- 保持现有深色基底和渐变气质，但减少面板之间的同质化
- 强化金额字号、间距和对比，做到一眼先看到数字，再看到标签
- 预算状态采用稳定的三段语义色：
  - 正常：冷绿
  - 接近上限：暖金
  - 超支：橘红
- 让图表、指标卡和列表拥有更统一的圆角、内边距和描边透明度

## 交互与空状态

- 如果没有任何月份数据，预算卡、排名卡和趋势卡显示统一空状态文案，不展示占位数字
- 如果某个分类只在一个月份出现，跨月排名仍正常显示单条结果，不额外提示错误
- 如果选中月份没有记录，默认退回最近一个有数据的月份；如果完全无数据，则保持当前输入月份并展示空状态

## 错误处理与一致性

- 所有预算与排名都基于内存汇总结果计算，不额外写回数据库，避免派生数据失真
- 金额继续以数值形式存储，展示层再做格式化
- 排名与汇总统一从同一次遍历生成，避免页面之间口径不一致

## 测试策略

优先为纯函数汇总层补测试，覆盖：

- 未超支月份的 `remaining` 和 `overspend`
- 超支月份的累计超支与平均每月超支
- 分类月内排名顺序
- 同分类跨月排名顺序
- 空数据和单月数据场景

界面侧至少验证：

- 预算状态在未超支与超支两种情况下文案和颜色切换正确
- 趋势页能在无数据、有数据、只有单分类数据时稳定渲染

## 实施边界

预计主要改动区域：

- `src/lib/ledgerSummary.ts`
- `src/components/Charts.tsx`
- `App.tsx`
- 视需要拆出少量预算/排名展示组件，避免 `App.tsx` 继续膨胀

本轮不调整数据库表结构；如果为了代码可读性新增纯展示组件，优先做小而清晰的拆分，不做与需求无关的重构。
